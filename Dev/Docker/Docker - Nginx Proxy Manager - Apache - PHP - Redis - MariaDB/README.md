# DOCKER - NGINX PROXY MANAGER - APACHE - PHP - REDIS - MARIADB

## 1. CREATE DOCKER NETWORK

CREATE A NETWORK TO CONNECT MULTIPLE CONTAINERS.

```bash
docker network create VMK_NPM_NETWORK_01
```

## 2. CREATE DOCKER COMPOSE - NGINX PROXY MANAGER

```bash
docker compose -p "nginx-proxy-manager" -f npm-docker-compose.yml up --build
```

## 3. CHANGE HOSTS FILE

```plain
127.0.0.1   YOUR-DOMAIN-NAME
```

## 4. CREATE SSL CERTIFICATE FILE

- CÀI ĐẶT PHẦN MỀM OPENSSL TRÊN WINDOWS
  - ĐỊA CHỈ TẢI OPENSSL -> [https://slproweb.com/download/Win64OpenSSL-3_6_1.msi](https://slproweb.com/download/Win64OpenSSL-3_6_1.msi)
  - SAU KHI CÀI ĐẶT THÌ MỞ FILE "SYSDM.CPL"
  - CHỌN THẺ "ADVANCED" NHẤN NÚT "ENVIRONMENT VARIABLES"
  - CHỌN "PATH" TRONG DANH SÁCH "SYSTEM VARIABLES" NHẤN NÚT "EDIT"
  - NHẤN NÚT "NEW" VÀ THÊM "C:\Program Files\OpenSSL-Win64\bin" VÀO DANH SÁCH
  - MỞ TERMINAL CHẠY LỆNH "openssl version" ĐỂ KIỂM TRA

- LỆNH TẠO SSL CERTIFICATE FILE

```bash
openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout example.key -out example.crt
```

## 5. SETUP SSL - NGINX PROXY MANAGER

CERTIFICATES -> ADD CERTIFICATE -> CUSTOM CERTIFICATE

- NAME: YOUR-DOMAIN-NAME
- CERTIFICATE: CHOOSE CERTIFICATE (CRT FILE)
- CERTIFICATE KEY: CHOOSE CERTIFICATE KEY (KEY FILE)

## 6. SETUP HOST - NGINX PROXY MANAGER

HOSTS -> PROXY HOSTS -> ADD PROXY HOST

- DETAILS
  - DOMAIN NAMES: YOUR-DOMAIN-NAME
  - SCHEME: HTTP
  - FORWARD HOSTNAME: WEB-APP-CONTAINER-NAME
  - FORWARD PORT: 80
  - BLOCK COMMON EXPLOITS: ON

- SSL
  - SSL CERTIFICATE: YOUR-DOMAIN-NAME
  - FORCE SSL: ON
  - HTTP/2 SUPPORT: ON
